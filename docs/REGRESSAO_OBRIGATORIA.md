# Checklist obrigatório de regressão

Uma atualização só pode ser aprovada quando todos os itens aplicáveis estiverem funcionando.

## Entrada e contas
- [ ] Página inicial elegante preservada.
- [ ] Escolha entre usuário comum e anunciante funcionando.
- [ ] Login de usuário funcionando.
- [ ] Login de anunciante funcionando.
- [ ] Login administrativo funcionando.
- [ ] Conta com papel duplo usuário + anunciante funcionando.
- [ ] Menus respeitam as permissões da conta.

## Feed
- [ ] Vídeos verticais 9:16 ocupam corretamente a tela.
- [ ] Deslizar para cima abre o próximo vídeo.
- [ ] Deslizar para baixo volta ao vídeo anterior.
- [ ] Curtir, salvar e compartilhar continuam funcionando.
- [ ] Produto aparece discretamente acima da navegação.
- [ ] Texto personalizado do botão do produto é respeitado.
- [ ] Modo Para Você e Modo Comprar funcionam.

## Shop
- [ ] Produtos internos são listados.
- [ ] Pesquisa e filtros funcionam.
- [ ] Página do produto abre corretamente.
- [ ] Carrinho mantém produtos e quantidades.
- [ ] Checkout não perde itens.
- [ ] Produto pode ter link externo opcional.

## Anunciante
- [ ] Upload de vídeo funciona.
- [ ] Upload de imagem funciona.
- [ ] Prévia do criativo aparece.
- [ ] Produto pode ser cadastrado com preço, estoque e comissão.
- [ ] Campanha pode ser vinculada a produto interno.
- [ ] Anunciante enxerga apenas os próprios dados.

## Usuário e carteira
- [ ] Pontos são registrados apenas em visualizações válidas.
- [ ] Saldo disponível e pendente permanecem separados.
- [ ] Solicitação de saque registra chave Pix e valor.
- [ ] Histórico não desaparece após recarregar.

## Afiliados
- [ ] Produto informa comissão.
- [ ] Usuário autorizado pode se afiliar.
- [ ] Link interno identifica produto e afiliado.
- [ ] Comissão só é gerada após evento válido de venda.

## Administração
- [ ] Admin não aparece para contas comuns.
- [ ] Campanhas podem ser aprovadas, pausadas ou recusadas.
- [ ] Saques podem ser revisados.
- [ ] Reservas e valores da plataforma permanecem consistentes.

## Dados e compatibilidade
- [ ] Dados existentes do localStorage continuam sendo reconhecidos.
- [ ] Nenhuma chave de armazenamento foi renomeada sem migração.
- [ ] Não há erros no console durante o fluxo principal.
- [ ] A aplicação funciona em tela de celular.
- [ ] CHANGELOG e versão foram atualizados.
